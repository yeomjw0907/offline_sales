# DESIGN.md

## 1. Visual Theme & Atmosphere

- 디자인 기준은 Notion에서 느껴지는 따뜻한 미니멀리즘이다.
- 제품은 문서 도구처럼 차갑지 않고, 업무 툴로서 신뢰감을 줘야 한다.
- 모집 화면은 부드럽고 친절해야 하고, 대시보드는 단정하고 명확해야 한다.
- 흰 배경, 넉넉한 여백, 낮은 대비의 경계선, 얕은 그림자를 사용한다.
- 과한 그라데이션, 과장된 3D 카드, 강한 네온 포인트는 사용하지 않는다.

## 3. Typography Rules

- 전체 인상은 과장되지 않은 에디토리얼 SaaS 느낌으로 간다.
- 제목은 차분하고 신뢰감 있게, 본문은 읽기 쉽게 유지한다.
- 한글 가독성을 우선한다.

| Role | Style |
|---|---|
| Display | 세리프 계열 또는 부드러운 제목용 폰트, 굵기 600 |
| H1 | 40-48px, line-height 1.15 |
| H2 | 28-32px, line-height 1.2 |
| H3 | 22-24px, line-height 1.25 |
| Body Large | 18px, line-height 1.65 |
| Body | 16px, line-height 1.65 |
| Caption | 13-14px, line-height 1.5 |
| Data / Code | 모노스페이스, 코드/추천인코드/정산값 노출용 |

## 4. Component Stylings

### Buttons

- 주 버튼은 짙은 먹색 배경 + 흰색 텍스트
- 라운드는 둥글지만 과하지 않게 `10-12px`
- hover는 그림자보다 명도 차로 처리
- CTA 문구는 짧고 직접적이어야 한다

### Inputs

- 흰 배경
- 얕은 회색 보더
- 포커스 시 검정이 아니라 차분한 회갈색 계열 outline
- placeholder는 너무 연하지 않게 한다

### Cards

- 카드 배경은 흰색
- 테두리는 연한 보더를 기본으로 사용
- 그림자는 최소화한다
- 통계 카드에는 숫자와 라벨의 위계를 명확히 둔다

### Badges

- 상태 배지는 채도가 낮은 배경색을 사용
- 성공/대기/비활성/지급완료 같은 운영 상태는 눈에 띄되 자극적이지 않게

### Tables & Lists

- 줄 간격 넉넉하게
- 행 hover는 연한 서브 배경으로 처리
- 열은 많아도 숨 막히지 않게 좌우 패딩 확보

## 5. Layout Principles

- 전체 레이아웃은 좁고 긴 문서형이 아니라, 넓은 여백의 운영형 화면으로 구성한다.
- 모집 화면은 한 섹션에 하나의 메시지만 강조한다.
- 대시보드는 상단에 핵심 숫자, 하단에 실적/자료/정산 정보를 배치한다.
- 관리자 화면은 검색, 필터, 표, 상세 패널 중심으로 구성한다.
- 모바일에서는 카드 스택, 데스크톱에서는 2~4열 카드 그리드를 사용한다.

## 6. Depth & Elevation

- 깊이는 낮게 유지한다.
- 기본은 보더 중심, 필요한 곳에만 아주 얕은 shadow를 사용한다.
- 모달/드로어는 카드보다 한 단계만 위로 올린다.

## 7. Do's and Don'ts

### Do

- 부드럽고 정돈된 여백을 유지한다.
- 정보 구조를 먼저 보여주고 장식은 최소화한다.
- 추천인 코드, 실적, 정산 숫자는 명확하게 강조한다.
- 모집 화면에서도 신뢰감과 실제 운영감이 느껴지게 한다.

### Don't

- 문서 사이트처럼 너무 밋밋하게 만들지 않는다.
- 대시보드를 마케팅 랜딩처럼 과장하지 않는다.
- 브랜드 포인트를 금색으로 처리하지 않는다.
- 과한 라운드, 떠 있는 카드, 유리질감, 네온 효과를 쓰지 않는다.

## 8. Responsive Behavior

- 모바일에서 신청 폼은 한 열로 정리한다.
- 모바일 대시보드는 핵심 수치 카드 2열 또는 1열로 자동 축소한다.
- 표는 모바일에서 카드형 리스트 또는 중요 컬럼 우선 노출 방식으로 전환한다.
- 터치 타겟은 최소 44px 이상을 유지한다.

## 9. Product-Specific Notes

- 파트너 홈 첫 화면은 `내 추천인 코드`, `이번 달 실적`, `예상 수수료`, `지급완료 금액`이 바로 보여야 한다.
- 영업자료 섹션은 문서 보관함이 아니라 빠른 실행 도구처럼 보여야 한다.
- 관리자 화면은 대량 작업보다 빠른 확인과 직접 입력에 최적화한다.
