# 운영 정책

## 1. 파트너 신청 및 승인

- 파트너는 카카오 로그인 후 신청서를 작성한다.
- 신청 필수 항목:
  - 이름
  - 전화번호
  - 이메일
  - 활동 지역
  - 유입경로
  - 활동 방식
- 선택 항목:
  - 간단한 소개 또는 활동 계획
- 신청 후 상태는 `승인대기`다.
- 관리자 승인 후 상태는 `활동중`으로 전환된다.
- 비활성 처리된 파트너는 신규 활동이 제한되지만 기존 실적은 유지된다.

## 2. 추천인 코드 정책

- 추천인 코드는 관리자 승인 시 자동 발급한다.
- 추천인 코드는 기본적으로 고정값이다.
- 코드 형식은 사람이 읽기 쉬운 영문/숫자 조합을 사용한다.
- 재발급은 최고관리자만 가능하다.
- 파트너는 코드와 QR을 대시보드에서 확인한다.

## 3. 실적 귀속 기준

- 실적 1건의 기준은 `매장 1개`다.
- 같은 사장님이 여러 매장을 운영하면 매장별로 별도 실적으로 인정한다.
- 추천인 코드는 Ready Talk 파일럿 시작 전에 입력되어야 한다.
- 실적 귀속 확정 시점은 `파일럿 시작`이다.
- 파일럿 시작 후에는 추천인 코드 변경이 불가하다.
- 이미 귀속된 매장은 다른 파트너에게 재귀속하지 않는다.
- 예외 수정은 최고관리자만 가능하다.

## 4. 중복 처리 정책

- 관리자 실적 입력 시 다음 항목을 기준으로 중복 의심을 확인한다.
  - 상호명
  - 연락처
  - 지역
- `상호명 + 연락처`가 동일하면 강한 중복 후보로 본다.
- `상호명 + 지역`이 유사하면 중복 의심 경고를 노출한다.
- 자동 차단보다 관리자 확인 중심으로 운영한다.

## 5. 실적 입력 정책

- 초기 MVP는 관리자 직접 입력 방식을 사용한다.
- 실적 등록 필수값:
  - 상호명
  - 연락처
  - 지역
  - 추천인 코드
  - 파일럿 시작일
- 입력 이력은 관리자 로그에 남긴다.

## 6. 정산 정책

- 정산 기준일은 매월 10일이다.
- 실적은 파일럿 시작 기준으로 인정한다.
- 정산 금액은 건당 2만원을 기준으로 계산한다.
- 지급 시 3.3%를 공제한다.
- 정산 상태는 `정산예정`, `지급완료`로 운영한다.
- 지급일과 실지급액을 기록한다.

## 7. 파트너 대시보드 정책

- 파트너는 다음 정보를 확인할 수 있다.
  - 내 추천인 코드
  - QR
  - 누적 실적 건수
  - 이번 달 실적 건수
  - 예상 수수료
  - 정산 예정 금액
  - 지급완료 금액
  - 최근 실적 내역
  - 영업자료
  - Ready Talk 바로가기 링크

## 8. 혜택 정책

- 자영업자가 추천인 코드를 입력하면 무료 파일럿 7일을 추가 제공한다.
- 운영 메시지는 `추천인 코드 입력 시 총 37일 무료`로 통일한다.

## 9. 개인정보 처리 정책

- 신청 단계와 정산 단계의 개인정보 수집은 분리한다.
- 신청 단계에서는 정산용 계좌정보를 받지 않는다.
- 정산 대상자가 생기면 별도 화면에서 계좌정보를 수집한다.
- 필수 동의:
  - 파트너 모집 및 운영
  - 실적 관리
  - 정산 처리
- 선택 동의:
  - 광고성 정보 수신
- 개인정보 처리방침은 별도 페이지로 공개한다.
- 민감한 정산 정보는 접근 권한을 제한한다.
- 주민등록번호 등 고위험 식별정보는 MVP 범위에서 제외하고, 필요 시 별도 검토 후 설계한다.
