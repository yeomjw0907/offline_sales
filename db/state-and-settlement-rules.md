# 상태값 및 정산 규칙

## 1. 파트너 상태

| 상태 | 설명 |
|---|---|
| `pending` | 신청 후 승인 대기 |
| `active` | 승인 완료, 활동 가능 |
| `inactive` | 활동 중지, 기존 실적 유지 |

### 전이 규칙

- `pending -> active`: 관리자 승인
- `active -> inactive`: 관리자 비활성 처리
- `inactive -> active`: 관리자 재활성화

## 2. 실적 상태

| 상태 | 설명 |
|---|---|
| `pilot_started` | 파일럿 시작 확정 |
| `settlement_ready` | 정산 대상 반영 완료 |
| `paid` | 정산 완료 |

### 전이 규칙

- 실적은 파일럿 시작일이 들어오면 `pilot_started`
- 월 정산 대상에 포함되면 `settlement_ready`
- 해당 정산이 지급완료되면 `paid`

## 3. 정산 상태

| 상태 | 설명 |
|---|---|
| `scheduled` | 정산예정 |
| `paid` | 지급완료 |

## 4. 정산 계산 규칙

- 건당 기준 금액: `20,000원`
- 원천 공제율: `3.3%`
- 실지급액 계산식:

```text
gross_amount = total_cases * 20,000
withholding_tax_amount = gross_amount * 0.033
net_amount = gross_amount - withholding_tax_amount
```

- 실제 구현 시 세금 계산은 반올림/절사 기준을 별도로 고정해야 한다.

## 5. 월 정산 로직

- 기준일: 매월 10일
- 한 달 단위로 파트너별 실적을 집계한다.
- 이미 다른 정산에 포함된 실적은 중복 정산하지 않는다.
- 지급완료 처리 시 지급일과 처리 담당자를 기록한다.

## 6. 귀속 규칙

- 실적은 `매장 1개 = 1건`
- 파일럿 시작 시점에 귀속 확정
- 파일럿 시작 후 추천인 코드 변경 불가
- 이미 귀속된 동일 매장은 다른 파트너에게 재귀속 불가
- 예외 수정은 최고관리자만 가능
